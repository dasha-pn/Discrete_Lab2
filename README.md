# Discrete_Lab2
## Розподіл роботи
- `client.py` — Шевчук Дарина.
- `server.py` — Сиротюк Віктор.

## Суть проєкту

Цей проєкт реалізує просту клієнт-серверну мережу для текстового чату з використанням асиметричного шифрування (RSA). Обмін повідомленнями між клієнтами відбувається через сервер, причому всі повідомлення шифруються і розшифровуються за допомогою відповідних відкритих і приватних ключів.

## Структура

- **`server.py`** — сервер приймає підключення клієнтів, виконує генерацію власного RSA-ключа, отримує публічні ключі клієнтів, дешифрує отримані повідомлення і надсилає їх іншим клієнтам у зашифрованому вигляді.
- **`client.py`** — клієнт підключається до сервера, передає своє ім’я, генерує власну пару RSA-ключів, обмінюється ключами із сервером, а потім надсилає і отримує зашифровані повідомлення.

## Суть RSA-шифрування

- Кожна сторона генерує власну пару ключів (відкритий та закритий ключ) з випадковими простими числами `p`, `q`.
- Обчислюється:
  - `n = p * q`
  - `phi = (p - 1) * (q - 1)`
  - `e` — невелике непарне число, взаємно просте з `phi`
  - `d = e⁻¹ mod phi` (модульно обернений)
- Повідомлення шифруються як `c = m^e mod n` і розшифровуються як `m = c^d mod n`, де `m` — ASCII-код символу.

## Логіка обміну повідомленнями

1. **Клієнт:**
   - Підключається до сервера і передає своє ім’я.
   - Отримує публічний ключ сервера.
   - Генерує свою пару RSA-ключів та надсилає серверу відкритий ключ.
   - Після цього запускає два потоки:
     - `write_handler`: шифрує текст і надсилає його серверу.
     - `read_handler`: отримує зашифровані повідомлення, дешифрує їх і виводить.

2. **Сервер:**
   - Приймає нові підключення, зберігає ім’я користувача та його відкритий ключ.
   - Дешифрує отримане повідомлення своїм приватним ключем.
   - Перешифровує для кожного клієнта з використанням їх публічного ключа та надсилає.

## Запуск

1. **Сервер:**
   ```
   python server.py
   ```

2. **Клієнт:**
   ```
   python client.py
   ```

## Пояснення імплементації

### `server.py`:
- При запуску створює RSA-пару ключів для сервера (використовуючи `randprime` з `sympy`).
- Чекає підключення клієнтів, зберігає їх сокети, імена та публічні ключі.
- Коли клієнт надсилає повідомлення:
  - Дешифрує повідомлення власним приватним ключем (`server_private_key`).
  - Перешифровує повідомлення публічними ключами інших клієнтів.
  - Розсилає його всім, окрім відправника.

### `client.py`:
- При запуску підключається до сервера, надсилає ім’я користувача.
- Отримує публічний ключ сервера.
- Генерує власну RSA-пару ключів (використовуючи `randprime`).
- Передає свій публічний ключ серверу.
- Запускає два паралельні потоки:
  - Один зчитує повідомлення з сокета, дешифрує приватним ключем і виводить у консоль.
  - Інший зчитує повідомлення з клавіатури, шифрує публічним ключем сервера і надсилає його.

Обмін повідомленнями відбувається в зашифрованому вигляді, забезпечуючи конфіденційність навіть при спілкуванні через сервер.

## Висновки

- Система демонструє базові принципи шифрування повідомлень у мережах.
- Використання асиметричної криптографії (RSA) дозволяє безпечно передавати дані між клієнтами, навіть без загального секретного ключа.
- Архітектура легко розширюється для підтримки додаткових функцій (наприклад, підписів чи автентифікації).
